# Portfolio Summary

## PowerShell Endpoint Automation Toolkit

This repository is a small public-facing sample of the kind of automation work I enjoy building for Windows environments.

My approach is straightforward:

- identify repetitive operational work
- convert that work into repeatable scripts
- collect useful output
- make troubleshooting easier for the next person
- keep the code readable and maintainable

The examples in this repository focus on three practical areas:

1. **Endpoint health checks**  
   Quick scripts that gather the most useful facts first.

2. **Deployment readiness**  
   Preflight checks that help avoid failed installs and reduce avoidable troubleshooting.

3. **Diagnostics collection**  
   Lightweight data gathering for troubleshooting Windows systems in a structured way.

## Public Sharing Standard

Any script prepared for public sharing should be sanitized:

- no organization-specific infrastructure names
- no internal paths or server references
- no secrets, tokens, or invitation codes
- no customer, student, or staff identifiers

## What I Want This Repository to Show

- practical systems thinking
- safe automation habits
- clean PowerShell structure
- clear operator-friendly output
- respect for privacy and environment separation
